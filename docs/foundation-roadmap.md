# 当前还缺什么

## 第一阶段必须补齐

- 学生、监护人、教师、账号角色绑定正式落库
- 学生档案补齐 `schoolName`、`className`、服务状态、缴费状态、服务起止时间
- 家长与多个学生的绑定关系落库
- 机构服务日历，明确每天是否开放用餐登记和作业辅导
- 每日作业模型，支持按学校、班级、日期分配
- 用餐记录和作业记录的真实查询、写入、修改接口
- 图片上传、文本备注、日期筛选的真实链路

## 第二阶段建议补齐

- 管理后台的批量维护能力：学生、缴费、服务时长、服务日历
- 管理后台的列表筛选、搜索、导出、审计日志
- 小程序的角色切换和多学生切换
- 家长查看后的回执、已读状态和异常提醒

## 第三阶段再考虑

- 到校 / 离校签出
- 家长补充反馈
- 学生自主查看
- 财务对账与课消

## 当前最合理的开发顺序

1. 先把学生、监护人、教师、服务有效期、机构服务日历建模
2. 再做教师端“每日作业 + 用餐 + 作业完成情况”录入闭环
3. 再做家长端按日期查看、多学生切换
4. 最后补管理后台批量维护和统计
