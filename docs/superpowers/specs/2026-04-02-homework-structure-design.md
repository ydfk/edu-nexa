# 每日作业与作业记录结构设计

## 目标

将每日作业从“单条文本记录”调整为“作业主表 + 作业内容子表”，并把作业记录收敛为“同一学生同一天同一科目只有一条整体记录”。

## 业务规则

- 同一个学校、同一个班级、同一天、同一个科目，只允许存在一份每日作业。
- 一份每日作业可以包含多条作业内容，按顺序展示和编辑。
- 同一个学生、同一天、同一个科目，只允许存在一条作业记录。
- 作业记录当前只记录整体完成状态、照片和备注。
- “部分完成”当前只作为整体状态，不细分到单条作业内容。
- 作业记录必须依附当天对应科目的每日作业，不能脱离作业主数据单独创建。

## 数据结构

### 每日作业主表

- 继续保留现有每日作业主表，唯一维度为 `school/class/date/subject`
- 继续保存教师、附件、备注等主信息
- 旧 `content` 字段作为兼容字段保留，用于历史数据和旧界面过渡

### 每日作业内容子表

- 新增独立子表，归属到每日作业主表
- 每条子内容至少包含：所属作业 ID、排序号、内容文本
- 前端录入和展示都以子内容数组为准

### 作业记录表

- 继续保留单表结构，不在本次拆记录子表
- 新增对每日作业主表的关联字段
- 新增明确科目字段，唯一维度为 `student/date/subject`
- 保留照片、备注、记录人等字段

## 前端交互

### 每日作业

- 录入页编辑的是一份作业，不是单条内容
- 主维度固定为学校、年级、班级、日期、科目
- 作业内容改为可增删的多条输入项
- 列表和面板都展示同一份作业下的多条内容

### 作业记录

- 新增面板视图，交互对齐用餐记录
- 按“学校/年级/班级 -> 学生 -> 当天科目作业”展示
- 每个学生每个科目只有一张记录卡片或一个记录入口
- 无对应每日作业时，不提供记录入口

## 兼容与迁移

- 旧每日作业数据保留主表记录，并将原 `content` 迁为一条默认子内容
- 旧作业记录按 `school/class/date/subject` 回填关联的每日作业主记录
- 如果旧数据缺少可唯一匹配的信息，保留旧记录但不强制回填错误关联
- 新接口返回中继续兼容 `content` 字段，值由子内容按换行拼接得出

## 接口边界

- 每日作业接口新增 `items` 数组输入输出
- `items` 至少一条且内容不能为空
- 作业记录接口要求提交科目，并校验存在对应每日作业
- 重复创建时返回明确业务错误

## 测试范围

- 每日作业唯一性校验
- 每日作业多条内容保存与读取
- 作业记录唯一性校验
- 作业记录依附每日作业校验
- 历史数据迁移兼容
