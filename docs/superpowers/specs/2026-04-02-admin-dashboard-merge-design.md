# 管理后台工作台与统计看板合并设计

## 目标

- 将管理后台现有的 `工作台` 与 `统计看板` 合并为单一首页入口。
- 删除空白的 `统计看板` 页面与侧边栏入口，只保留 `工作台`。
- 基于当前已经接入的真实业务数据，提供一组轻量、可信、可直接用于运营查看的总览统计。

## 当前现状

- 首页 `工作台` 已接入学校、学生、用餐记录、作业记录等真实接口。
- `统计看板` 页面当前为空页面，仅展示“无”，没有实际业务价值。
- `工作台` 中存在一张“近 6 个月趋势”图，但数据为前端写死的模拟数据，不符合真实统计要求。

## 设计结论

- 合并后仅保留 `/` 作为首页入口，页面名称继续使用 `工作台`。
- 删除 `/statistics` 路由以及侧边栏中的 `统计看板` 菜单项。
- 首页改造成“运营总览”页面，面向 `admin`、`teacher` 展示真实统计；`guardian` 保持精简视图。

## 页面结构

### 1. 核心指标

首页顶部展示 5 张指标卡：

- 学校数量
- 学生总数
- 今日用餐完成数
- 今日作业提交数
- 本月缴费金额

其中“本月缴费金额”只在 `admin`、`teacher` 视图中显示。

### 2. 近 7 天趋势

展示最近 7 天每日的两项趋势：

- 用餐完成数
- 作业提交数

该区域替换当前首页中的“近 6 个月趋势”模拟图，不再使用写死数据。

### 3. 学校概览

按学校输出轻量汇总列表或表格，每个学校展示：

- 学生数
- 近 7 天用餐完成数
- 近 7 天作业提交数

该区域仅面向 `admin`、`teacher` 展示。

### 4. 最近记录

保留现有“最近用餐”和“最近作业”两块内容，继续展示最近 5 条真实记录。

## 数据来源与统计口径

### 学校数量

- 数据来源：`fetchSchools({ status: "active" })`
- 统计口径：仅统计状态为 `active` 的学校

### 学生总数

- 数据来源：`fetchStudents()`
- 统计口径：
  - `admin`、`teacher` 统计全部可见学生
  - `guardian` 仅统计其关联学生

### 今日用餐完成数

- 数据来源：`fetchMealRecords()`
- 统计口径：`serviceDate === 今天` 且 `status === "completed"`

### 今日作业提交数

- 数据来源：`fetchHomeworkRecords()`
- 统计口径：`serviceDate === 今天` 且 `status` 为 `completed` 或 `partial`
- 说明：沿用现有首页的作业完成口径，避免合并后语义变化

### 本月缴费金额

- 数据来源：`fetchPaymentRecords()`
- 统计口径：
  - `paidAt` 落在当前自然月内
  - 金额累计字段为 `paymentAmount`
  - 本次版本不扣减 `refundAmount`
- 说明：本指标定义为“本月缴费金额”，不是“本月净收入”

### 近 7 天趋势

- 数据来源：`fetchMealRecords()`、`fetchHomeworkRecords()`
- 统计口径：
  - 时间范围为今天往前 6 天，共 7 天
  - 用餐统计仅包含 `completed`
  - 作业统计包含 `completed` 与 `partial`

### 学校概览

- 数据来源：学生、用餐记录、作业记录三类真实数据
- 统计口径：
  - 以学校为粒度聚合
  - 展示该校学生总数、近 7 天用餐完成数、近 7 天作业提交数
  - 首版采用前端聚合，不新增后端统计接口

## 角色差异

### admin / teacher

展示完整的运营总览页面，包括：

- 核心指标 5 张卡片
- 近 7 天趋势
- 学校概览
- 最近记录

### guardian

继续展示精简版工作台，仅展示与自己相关的数据：

- 关联学生数
- 用餐记录数
- 作业记录数
- 最近用餐
- 最近作业

`guardian` 不展示学校概览与本月缴费金额，也不需要展示面向全局的运营趋势。

## 路由与导航调整

- 删除 `main.tsx` 中的 `/statistics` 路由定义
- 删除 `sidebar-data.ts` 中的 `统计看板` 菜单项
- 同步更新依赖侧边栏结构的测试断言

## 实现边界

- 主要改造文件为 `apps/admin/src/pages/dashboard.tsx`
- 如果首页聚合逻辑明显增多，可拆分出一个小型 dashboard helper 模块承载纯函数统计逻辑
- 本次不新增后端接口，不调整其他业务页行为，不做无关重构

## 测试范围

- 更新侧边栏测试，确认 `统计看板` 不再出现
- 为首页统计聚合补充测试，至少覆盖：
  - 本月缴费金额计算
  - 近 7 天趋势聚合
  - 学校概览聚合
- 优先将聚合逻辑提取为纯函数后再测试，降低页面测试复杂度

## 非目标

- 不在本次改造中引入复杂筛选器、日期范围切换或钻取分析
- 不将本月缴费金额改造成净收入口径
- 不为统计能力单独新增新的业务入口
