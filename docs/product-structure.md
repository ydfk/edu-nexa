# 产品结构草案

## 核心定位

学栖 · EduNexa 当前先聚焦托管机构晚辅场景，只做三条主线：

- 教师记录每日用餐情况
- 教师记录每日作业情况
- 监护人按日期查看自己孩子的反馈

## 角色与登录

- 管理员：配置机构日历、首页内容、账号权限、基础档案
- 教师：维护学生、监护人、服务状态，记录用餐和作业，按学校和班级编辑每日作业
- 监护人：查看自己名下一个或多个学生的每日用餐和作业反馈
- 学生：默认只作为服务对象存在，后续可扩展自主查看

说明：

- 管理员、教师、监护人可能是同一人
- 当前统一按手机号账号体系设计
- 管理端使用手机号 + 密码登录
- 小程序使用微信登录态 + 获取手机号授权

## 关键业务约束

- 家长可能绑定多个学生，小程序所有查询必须支持按学生切换或聚合展示
- 教师在小程序中需要能查看机构内全部学生，并按日期编辑当天用餐与作业记录
- 学生档案必须包含 `schoolName` 和 `className` 字段，用于作业分发
- 班级字段当前只作为学生属性和作业分组条件，不作为独立运营模块
- 管理员需要配置具体哪一天提供晚辅用餐、哪一天提供作业辅导
- 缴费不是单纯布尔值，还需要记录缴费时间和服务有效期

## 小程序信息架构

### 1. 首页

- 机构介绍
- 可配置 banner、图片、公告、服务说明
- 今日是否提供用餐 / 作业辅导
- 家长端显示孩子数量、最近服务记录入口
- 教师端显示待补录数量、今日待处理入口

### 2. 用餐

- 支持日期筛选
- 家长查看自己名下学生在指定日期的用餐记录
- 教师查看机构内全部学生在指定日期的用餐记录
- 教师可编辑用餐状态、备注、图片
- 记录必须保留学生维度和日期维度

### 3. 作业

- 支持日期筛选
- 按学生展示当天作业完成情况
- 当前优先状态可先收敛为：`已完成`、`完成一部分`
- 支持图片和文字反馈
- 家长查看，教师编辑

### 4. 每日作业编辑

- 教师按 `学校 -> 班级 -> 日期` 维护当天作业内容
- 学生根据自身 `schoolName + className` 自动关联当日作业
- 作业记录分两层：
- 第一层是班级当天作业内容
- 第二层是学生当天完成情况和反馈

### 5. 我的

- 当前账号角色信息
- 绑定学生列表
- 服务有效期与缴费状态摘要
- 教师常用设置入口

## 数据主线

1. 账号 -> 角色 -> 校区
2. 学生 -> 学校 -> 班级 -> 监护人
3. 学生 -> 服务有效期 -> 缴费记录
4. 日期 -> 机构服务日历 -> 是否有用餐 / 作业辅导
5. 学校 + 班级 + 日期 -> 每日作业
6. 学生 + 日期 -> 用餐记录 / 作业完成记录

## 双端职责建议

### 小程序必须有

- 首页内容展示
- 家长查看多学生的用餐和作业记录
- 教师按日期录入和修改用餐记录
- 教师按日期录入和修改学生作业完成情况
- 教师按学校和班级编辑每日作业
- 教师快速查看和修改学生服务状态

### 建议小程序和管理后台都实现

- 用餐记录列表和日期筛选
- 作业记录列表和日期筛选
- 学生详情页
- 监护人绑定关系查看
- 学生服务有效期和缴费状态查看
- 每日作业查看

说明：

- 这些功能在小程序里是为了移动端即时处理
- 在管理后台里再实现一次是为了批量筛选、批量编辑、导出和审计

### 更适合以管理后台为主

- 首页 banner、介绍文案、图片素材配置
- 学生、监护人、教师档案的批量维护
- 学校和班级字段规则维护
- 缴费记录、服务时长、服务起止时间维护
- 哪一天有晚辅用餐、哪一天有作业辅导的机构日历维护
- 账号角色绑定、权限管理
- 数据统计、导出、异常回查

说明：

- 小程序仍可保留单条快捷修改
- 正式维护入口应放在后台，否则教师在手机上维护缴费和日历会非常低效
