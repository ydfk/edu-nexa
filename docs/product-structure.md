# 产品结构草案

## 角色

- 管理员：配置校区、账号、权限、模板
- 教师：记录学生当日晚餐和作业完成情况
- 监护人：查看孩子当日晚餐和作业反馈
- 学生：作为服务对象存在，后续可扩展到自主查看

说明：

- 管理员、教师、监护人可能是同一人
- 当前统一按手机号账号体系设计
- 管理端使用手机号 + 密码登录
- 小程序使用微信登录态 + 获取手机号授权

## 当前优先模块

### 1. 基础档案

- 校区
- 学生
- 监护人
- 教师 / 员工
- 账号与角色绑定

### 2. 晚辅执行

- 到校签到
- 用餐登记
- 作业完成情况
- 作业备注与图片补充
- 离校确认

### 3. 家长查看

- 晚餐状态
- 作业完成情况
- 异常提醒
- 每日汇总

### 4. 运营后台

- 今日概览
- 校区维度看板
- 学生维度明细
- 教师工作量
- 家长查看回执

## 建议的数据主线

1. 账号 -> 角色 -> 校区
2. 学生 -> 监护人
3. 学生 -> 当日服务记录
4. 当日服务记录 -> 用餐记录 / 作业记录 / 家长查看
